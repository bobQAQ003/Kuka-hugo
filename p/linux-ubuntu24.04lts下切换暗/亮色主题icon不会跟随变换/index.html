[Linux] Ubuntu24.04lts下切换暗/亮色主题，icon不会跟随变换

2026-04-13

桌面是 GNOME，用了 gnome-tweaks 插件更换图标和主题，笨办法，没找到其他解决方案。

问题描述

在当前主题（暗色）进行切换，原本的图标（Reversal-blue-dark）图标会变成默认；之后即使再切回，图标也不会自动恢复。

原因

GNOME 中下面这几项设置是彼此独立的：

  • org.gnome.desktop.interface color-scheme
  • org.gnome.desktop.interface gtk-theme
  • org.gnome.desktop.interface icon-theme

也就是说，切换亮色/暗色，只会修改 color-scheme，但 GNOME 不会自动根据亮暗状态，再把图标主题改回去。

解决思路

既然系统不会自动改回去，那最直接的办法就是：

  • 监听主题相关设置变化；
  • 每次变化后都检查当前图标主题；
  • 如果不是 Reversal-blue，就立刻改回 Reversal-blue；

脚本

监听脚本

路径：~/.local/bin/force-reversal-blue-icons.sh

作用：

  • 启动时先立即纠正一次图标主题
  • 然后持续监听 GNOME 的接口设置变化
  • 一旦检测到和主题相关的键发生变化，就重新把图标主题设为 Reversal-blue
 1#!/usr/bin/env bash
 2
 3set -euo pipefail
 4
 5# 定义目标主题和监听范围
 6TARGET_THEME="Reversal-blue"                    # 强制保持的目标图标主题
 7SCHEMA="org.gnome.desktop.interface"            # 监听的 gsettings schema
 8KEYS=("color-scheme" "gtk-theme" "icon-theme")
 9
10# 读取当前图标主题，若当前值不是 `Reversal-blue` 就改
11ensure_theme() {
12  local current
13  current="$(gsettings get "$SCHEMA" icon-theme | tr -d "'")"
14
15  if [[ "$current" != "$TARGET_THEME" ]]; then
16    gsettings set "$SCHEMA" icon-theme "$TARGET_THEME"
17  fi
18}
19
20# 脚本一启动就先执行一次检查
21main() {
22  ensure_theme
23
24  # `gsettings monitor org.gnome.desktop.interface` 持续输出这个 schema 下所有键的变化。
25  gsettings monitor "$SCHEMA" |
26    while IFS= read -r line; do
27      for key in "${KEYS[@]}"; do
28        if [[ "$line" == "$key"* ]]; then
29          ensure_theme
30          break
31        fi
32      done
33    done
34}
35
36main "$@"

自启动文件

路径：~/.config/autostart/force-reversal-blue-icons.desktop

作用：

  • 在登录 GNOME 会话时自动启动脚本
1[Desktop Entry]
2Type=Application
3Name=Force Reversal Blue Icons
4Comment=Keep the GNOME icon theme fixed to Reversal-blue
5Exec=~/.local/bin/force-reversal-blue-icons.sh
6X-GNOME-Autostart-enabled=true
7Terminal=false
  • Type=Application：这是一个可启动程序
  • Exec=...：登录后执行这个脚本
  • X-GNOME-Autostart-enabled=true：启用自动启动
  • Terminal=false：后台运行，不弹出终端窗口
